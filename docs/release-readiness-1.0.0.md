# Semantic Release 1.0.0 Readiness Checklist

Use this checklist before promoting `1.0.0`.

## Dry-run validation

- On `develop`, run `npm run release:dry:develop` and confirm:
  - a `beta` prerelease is proposed
  - release notes include expected commits
- On `main`, run `npm run release:dry:main` and confirm:
  - a stable release is proposed
  - release notes and bump type match expectations

## GitHub Actions validation

- Confirm `.github/workflows/release.yml` succeeds on both `develop` and `main`.
- Confirm `.github/workflows/commitlint.yml` reports guidance warnings without blocking PR merge.
- Confirm `CHANGELOG.md`, release tags, and GitHub Releases are generated by automation.

## 1.0.0 go-live checks

- Ensure all intended 1.0.0 features/fixes are merged to `main`.
- Ensure release notes for `1.0.0` are accurate and user-facing.
- Ensure branch protections require CI quality gates before merge to `main`.
- Merge to `main` and let semantic-release publish `v1.0.0`.

## Post-1.0.0 tasks

- Remove pre-1.0 fallback message rules from `.releaserc.cjs`.
- Switch commitlint from non-blocking guidance to required PR enforcement.
- Keep prerelease flow on `develop` for validation (`beta` channel).
